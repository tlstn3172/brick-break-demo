# PRD: Neon Breaker - 벽돌 깨기 게임

## 1. 프로젝트 개요

### 1.1 프로젝트 명
**Neon Breaker** - 네온 스타일의 모던한 벽돌 깨기 웹 게임

### 1.2 목적
클래식한 벽돌 깨기 게임을 현대적인 네온 디자인과 부드러운 사용자 경험으로 재해석하여, 모바일 및 데스크톱 환경에서 즐길 수 있는 웹 기반 캐주얼 게임을 제공합니다.

### 1.3 타겟 사용자
- 캐주얼 게임을 즐기는 모든 연령대의 사용자
- 모바일 및 데스크톱 브라우저 사용자
- 짧은 시간에 즐길 수 있는 게임을 찾는 사용자

- 네온 글로우 효과 (텍스트, 버튼, 게임 요소)

- 그라데이션 및 블러 효과를 활용한 깊이감

### 3.2 게임 화면 (Gameplay Screen)

![게임 화면](C:\Users\WIN\.gemini\antigravity\brain\951c59ee-0758-457c-b6e9-36641704b4fe\design_gameplay.png)

#### UI 구성 요소

**헤더 (HUD)**
- **일시정지 버튼** (좌측)
  - Pause 아이콘
  - 원형 버튼
  
- **레벨 표시** (중앙)
  - "LEVEL" 라벨
  - 현재 레벨 숫자 (큰 폰트)
  
- **생명 표시** (우측)
  - 하트 아이콘 (최대 3개)
  - 채워진 하트 / 빈 하트로 구분

**점수 표시**
- 현재 점수를 큰 숫자로 표시
- 네온 글로우 효과
- "Current Score" 라벨

**게임 영역**
- **벽돌 그리드**
  - 6열 그리드 레이아웃
  - 벽돌 타입별 색상:
    - 강한 벽돌: 보라색 그라데이션 (`purple-600` to `purple-400`)
    - 중간 벽돌: 시안/블루 그라데이션 (`cyan-500` to `blue-500`)
    - 약한 벽돌: 틸/에메랄드 그라데이션 (`teal-400` to `emerald-400`)
  - 깨진 벽돌: 반투명 회색
  - 각 벽돌은 네온 그림자 효과

- **공 (Ball)**
  - 흰색 원형
  - 글로우 효과
  - 펄스 애니메이션

- **파티클 효과**
  - 충돌 시 작은 파티클 생성
  - Primary 컬러 및 흰색

- **패들 (Paddle)**
  - Primary 컬러
  - 라운드 캡슐 형태
  - 네온 그림자 효과
  - 내부 디테일 (흰색 라인)

- **궤적 힌트**
  - 패들에서 나오는 반투명 라인
  - 공의 예상 경로 표시

**터치 컨트롤 영역**
- 하단에 점선 테두리 영역
- "Touch Control Zone" 안내 텍스트
- "Slide anywhere to move paddle" 설명
- 터치 아이콘 (애니메이션)
- iOS 홈 인디케이터 영역 확보

#### 게임 로직 요구사항

**기본 게임플레이**
- 공은 패들에서 발사되어 벽돌을 깨뜨림
- 공이 화면 하단으로 떨어지면 생명 1개 감소
- 모든 벽돌을 깨면 다음 레벨로 진행
- 생명이 0이 되면 게임 오버

**벽돌 시스템**
- 벽돌 타입별 내구도:
  - 약한 벽돌 (틸): 1회 충돌로 파괴
  - 중간 벽돌 (블루): 2회 충돌로 파괴
  - 강한 벽돌 (보라): 3회 충돌로 파괴
- 벽돌 파괴 시 점수 획득
- 파괴된 벽돌은 페이드아웃 애니메이션

**점수 시스템**
- 벽돌 파괴 시 점수 획득
  - 약한 벽돌: 10점
  - 중간 벽돌: 20점
  - 강한 벽돌: 30점
- 연속 파괴 시 콤보 보너스
- 레벨 클리어 보너스

**물리 시스템**
- 공의 속도는 레벨이 올라갈수록 증가
- 패들의 어느 부분에 맞느냐에 따라 반사 각도 변화
- 벽면 및 천장에서 반사

**컨트롤**
- 마우스/터치로 패들 좌우 이동
- 터치 컨트롤 영역 어디서나 슬라이드 가능
- 패들은 화면 경계를 벗어나지 않음

**일시정지 기능**
- 일시정지 버튼 클릭 시 게임 정지
- 일시정지 모달 표시:
  - 계속하기
  - 다시 시작
  - 메인 메뉴로

## 8. 개발 우선순위

### Phase 1: MVP (Minimum Viable Product)
1. 기본 게임 화면 구현 (시작, 게임플레이, 게임 오버)
2. 핵심 게임 로직 (공, 패들, 벽돌 충돌)
3. 점수 및 생명 시스템
4. 레벨 시스템 (최소 5개 레벨)
5. 로컬 스토리지 연동

### Phase 2: 폴리싱
1. 애니메이션 및 효과 개선
2. 사운드 추가
3. 추가 레벨 디자인
4. 성능 최적화
5. 크로스 브라우저 테스트

### Phase 3: 확장
1. 파워업 시스템
2. 추가 게임 모드
3. 온라인 기능
4. 모바일 앱 변환 (PWA)
